В качестве основы для лабораторной работы использовался проект из лабораторной работы 5. Исходный код проекта размещён в репозитории GitHub: https://github.com/danluki/qrgen-lab.

Для автоматизации процессов сборки и проверки был настроен CI/CD pipeline с использованием GitHub Actions. Pipeline автоматически запускается при изменении кода в репозитории, в частности при push и pull request.

В рамках pipeline выполняются следующие этапы:

Получение актуальной версии проекта из репозитория.
Установка зависимостей backend и frontend частей приложения.
Сборка проекта.
Запуск тестов и проверки работоспособности.
Получение итогового статуса выполнения pipeline.
При корректном состоянии проекта pipeline завершается успешно. При наличии ошибок в коде, сборке или тестах pipeline завершается с ошибкой, что позволяет оперативно выявлять проблемы после внесения изменений.
